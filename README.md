## 参加者向けの流れ

1. Welcome画面
2. 名前の入力
3. 撮影希望の選択
4. 内容の確認
5. 回答の保存
6. 3秒後、または「次の方へ」で最初の画面へ戻る

## 撮影希望の選択肢

1. 写真OK
2. 顔が写っていなければOK
3. みんなと写っていればOK
4. ボケていたり、身体の一部分ならOK
5. 今回は撮影を見送り

選択肢は、緑から黄色、オレンジ、薄い赤、赤へ段階的に色分けしています。顔アイコンと紫色は使用していません。

## 主な機能

- イベントごとの回答管理
- 回答の検索・編集・削除
- イベントの作成・終了・再開・削除
- CSV出力
- JSONバックアップと復元
- PWA対応
- PC・iPhone対応
- キーボード操作と「動きを減らす」設定への配慮
- 名前や選択が未入力の場合の画面内案内

回答データは、使用中のブラウザ内に保存されます。端末やブラウザを変更した場合、データは自動的には引き継がれません。必要に応じて管理画面からバックアップを保存してください。

## 管理画面

Welcome画面右上の歯車を約2秒長押しすると、管理者ログイン画面が開きます。

初期パスコード：`1234`

実際の運用前に、`script.js` 内のパスコードを変更してください。

## GitHub Pagesへの公開

1. ZIPを解凍します。
2. 中の全ファイルと `assets` フォルダを、GitHubリポジトリのルートへアップロードします。
3. GitHub Pagesを有効にします。
4. 更新後に古い画面が残る場合は、PCでは強制再読み込み、iPhoneではホーム画面版の再追加を行います。
